深入理解浏览器回流与重绘：原理与优化实践

发表于2024|更新于2026|基础知识
|字数总计:1.8k|阅读时长:7分钟

浏览器的回流（Reflow）和重绘（Repaint）是前端性能优化中的重要概念。理解它们的工作原理，对于编写高性能的网页应用至关重要。

一、浏览器的渲染过程

在讨论回流和重绘之前，我们需要先了解浏览器的渲染流程：

DOM树 → CSSOM树 → 渲染树 → 布局 → 绘制 → 合成

1.1 渲染流程详解

  1. 构建 DOM 树：解析 HTML，生成 DOM 树
  2. 构建 CSSOM 树：解析 CSS，生成 CSSOM 树
  3. 生成渲染树（Render Tree）：DOM 树与 CSSOM 树合并
  4. 布局（Layout/Reflow）：计算元素的位置和大小
  5. 绘制（Paint/Repaint）：将元素绘制到屏幕上
  6. 合成（Composite）：将多个图层合并显示

二、什么是回流（Reflow）

2.1 定义

回流是指浏览器重新计算元素的位置和几何属性，重新构建渲染树的过程。当页面的布局发生改变时，浏览器需要重新计算元素的位置和大小。

2.2 触发回流的情况

以下操作会触发回流：

// 1. 改变窗口大小
window.resizeTo(1000, 800)

// 2. 改变字体大小
document.body.style.fontSize = '20px'

// 3. 添加或删除元素
const div = document.createElement('div')
document.body.appendChild(div)

// 4. 改变元素尺寸
element.style.width = '200px'
element.style.height = '100px'
element.style.padding = '10px'
element.style.margin = '20px'
element.style.border = '1px solid #ccc'

// 5. 改变元素位置
element.style.position = 'absolute'
element.style.top = '100px'
element.style.left = '200px'

// 6. 读取某些属性或调用某些方法
const offsetWidth = element.offsetWidth
const offsetHeight = element.offsetHeight
const scrollTop = element.scrollTop
const clientWidth = element.clientWidth
const rect = element.getBoundingClientRect()
window.getComputedStyle(element)

2.3 常见触发回流的属性和方法

属性/方法 说明
offsetTop/Left/Width/Height 元素偏移量和尺寸
scrollTop/Left/Width/Height 滚动相关尺寸
clientTop/Left/Width/Height 客户区尺寸
getBoundingClientRect() 获取元素边界矩形
getComputedStyle() 获取计算后的样式
scrollIntoView() 滚动到可视区域
scrollTo() 滚动到指定位置

三、什么是重绘（Repaint）

3.1 定义

重绘是指元素的外观发生改变，但布局没有改变，浏览器只需重新绘制元素的过程。重绘的代价比重流小得多。

3.2 触发重绘的情况

以下操作只会触发重绘，不会触发回流：

// 改变颜色
element.style.color = 'red'
element.style.backgroundColor = '#fff'

// 改变边框颜色
element.style.borderColor = 'blue'

// 改变可见性
element.style.visibility = 'hidden'

// 改变背景
element.style.backgroundImage = 'url(...)'

// 改变阴影
element.style.boxShadow = '0 0 10px rgba(0,0,0,0.5)'

// 改变轮廓
element.style.outline = '1px solid red'

四、回流与重绘的关系

4.1 关系图

回流一定重绘，重绘不一定回流
操作 回流 重绘
改变元素尺寸/位置 ✅ ✅
改变元素颜色 ❌ ✅
改变字体大小 ✅ ✅
改变背景颜色 ❌ ✅
添加/删除元素 ✅ ✅

4.2 性能影响

// 回流的性能消耗远大于重绘
// 回流成本 = 计算布局 + 绘制
// 重绘成本 = 绘制

// 回流影响范围
// 全局回流：整个页面布局重新计算
// 局部回流：只影响部分元素

五、回流与重绘的优化策略

5.1 批量修改样式

// ❌ 不好的写法 - 触发多次回流
element.style.width = '100px'
element.style.height = '100px'
element.style.margin = '10px'
element.style.padding = '10px'

// ✅ 好的写法 - 只触发一次回流
// 方式1：使用 cssText
element.style.cssText = 'width: 100px; height: 100px; margin: 10px; padding: 10px;'

// 方式2：添加 CSS 类
element.classList.add('active')

// 方式3：一次性修改多个样式
Object.assign(element.style, {
  width: '100px',
  height: '100px',
  margin: '10px',
  padding: '10px'
})

5.2 批量修改 DOM

// ❌ 不好的写法 - 每次插入都触发回流
const list = document.getElementById('list')
for (let i = 0; i < 100; i++) {
  const item = document.createElement('li')
  item.textContent = `Item ${i}`
  list.appendChild(item) // 每次都触发回流
}

// ✅ 好的写法 - 使用 DocumentFragment
const fragment = document.createDocumentFragment()
for (let i = 0; i < 100; i++) {
  const item = document.createElement('li')
  item.textContent = `Item ${i}`
  fragment.appendChild(item)
}
list.appendChild(fragment) // 只触发一次回流

// ✅ 好的写法 - 使用 innerHTML
let html = ''
for (let i = 0; i < 100; i++) {
  html += `<li>Item ${i}</li>`
}
list.innerHTML = html

5.3 使用 display:none 隐藏元素

// 对隐藏元素的修改不会触发回流和重绘
element.style.display = 'none'

// 进行大量 DOM 操作
element.style.width = '100px'
element.style.height = '100px'
element.style.margin = '10px'

// 显示元素 - 只触发一次回流
element.style.display = 'block'

5.4 避免频繁读取布局属性

// ❌ 不好的写法 - 强制同步布局（Layout Thrashing）
for (let i = 0; i < 100; i++) {
  const width = element.offsetWidth // 读取触发回流
  element.style.width = width + 10 + 'px' // 写入触发回流
}

// ✅ 好的写法 - 先读取，后写入
const width = element.offsetWidth // 读取一次
for (let i = 0; i < 100; i++) {
  elements[i].style.width = width + 10 + 'px'
}

5.5 使用 transform 代替位置属性

// ❌ 不好的写法 - 触发回流
element.style.left = '100px'
element.style.top = '100px'

// ✅ 好的写法 - 只触发合成，不回流不重绘
element.style.transform = 'translate(100px, 100px)'

5.6 使用 opacity 代替 visibility

// visibility: hidden 触发重绘
element.style.visibility = 'hidden'

// opacity: 0 只触发合成
element.style.opacity = '0'

5.7 使用 CSS3 硬件加速

.optimized-element {
  /* 开启 GPU 加速 */
  transform: translateZ(0);
  /* 或 */
  will-change: transform;
}

// will-change 提示浏览器将要发生变化
element.style.willChange = 'transform'

5.8 使用 requestAnimationFrame

// ❌ 不好的写法
function update() {
  element.style.width = element.offsetWidth + 10 + 'px'
  element.style.height = element.offsetHeight + 10 + 'px'
}

// ✅ 好的写法 - 使用 requestAnimationFrame 批量更新
function update() {
  requestAnimationFrame(() => {
    element.style.width = element.offsetWidth + 10 + 'px'
    element.style.height = element.offsetHeight + 10 + 'px'
  })
}

六、使用 Chrome DevTools 分析

6.1 Performance 面板

  1. 打开 Chrome DevTools（F12）
  2. 选择 Performance 面板
  3. 点击录制按钮
  4. 执行操作
  5. 停止录制，分析结果

6.2 Layers 面板

查看元素的图层信息，确认是否使用了硬件加速。

6.3 Rendering 面板

开启以下选项可以可视化回流和重绘：

  • Paint flashing：绿色高亮重绘区域
  • Layout Shift Regions：蓝色高亮布局偏移
  • Layer borders：显示图层边框

七、现代浏览器优化

现代浏览器通过以下方式优化回流：

7.1 批量更新

浏览器会将多次回流操作合并成一次执行：

// 浏览器优化：这些操作会被批量处理
element.style.width = '100px'
element.style.height = '100px'
element.style.margin = '10px'

7.2 异步布局

// 读取操作会强制同步布局
const width = element.offsetWidth // 立即计算布局

7.3 增量布局

只重新计算受影响的部分，而不是整个页面。

八、总结

8.1 最佳实践

  1. 批量修改样式：使用 cssText、CSS 类或 Object.assign
  2. 批量修改 DOM：使用 DocumentFragment 或 innerHTML
  3. 避免强制同步布局：先读取后写入，避免读写交替
  4. 使用 CSS3 特性：transform、opacity、will-change
  5. 隐藏复杂动画：使用 display:none 进行批量操作
  6. 使用 requestAnimationFrame：合理安排布局计算时机

8.2 性能对比

性能消耗：回流 > 重绘 > 合成

优化优先级：
1. 避免回流
2. 减少重绘
3. 使用合成
