JavaScript深拷贝完全指南：从浅拷贝到终极实现

发表于2023|更新于2026|前端基础
|字数总计:3.1k|阅读时长:14分钟

前言：为什么需要深拷贝

在 JavaScript 开发中，我们经常需要复制对象。但 JavaScript 中的数据类型分为原始类型和引用类型：

  • 原始类型：Number、String、Boolean、null、undefined、Symbol、BigInt - 存储在栈中
  • 引用类型：Object - 存储在堆中

直接赋值引用类型只是复制了引用地址，修改新对象会影响原对象。因此，我们需要深拷贝来创建一个完全独立的副本。

一、浅拷贝 vs 深拷贝

1.1 赋值、浅拷贝、深拷贝的区别

类型 修改新对象是否影响原对象 复制层级
直接赋值 ✅ 影响 只复制引用地址
浅拷贝 ❌ 不影响第一层
✅ 影响嵌套层		 只复制第一层
深拷贝 ❌ 不影响 递归复制所有层级

1.2 示例对比

// 直接赋值
const obj1 = { name: 'Alice', info: { age: 18 } };
const obj2 = obj1;
obj2.name = 'Bob';
obj2.info.age = 20;
console.log(obj1); // { name: 'Bob', info: { age: 20 } } - 被修改了！

// 浅拷贝
const obj3 = { name: 'Alice', info: { age: 18 } };
const obj4 = { ...obj3 }; // 或 Object.assign({}, obj3)
obj4.name = 'Bob';
obj4.info.age = 20;
console.log(obj3); // { name: 'Alice', info: { age: 20 } } - 嵌套对象被修改了！

// 深拷贝
const obj5 = { name: 'Alice', info: { age: 18 } };
const obj6 = JSON.parse(JSON.stringify(obj5));
obj6.name = 'Bob';
obj6.info.age = 20;
console.log(obj5); // { name: 'Alice', info: { age: 18 } } - 完全不受影响！

二、浅拷贝方法汇总

2.1 Object.assign()

const obj = { a: 1, b: { c: 2 } };
const cloneObj = Object.assign({}, obj);

cloneObj.a = 10;
cloneObj.b.c = 20;

console.log(obj.a);     // 1 - 第一层未受影响
console.log(obj.b.c);   // 20 - 嵌套层被影响

2.2 扩展运算符 (…)

const obj = { a: 1, b: { c: 2 } };
const cloneObj = { ...obj };

cloneObj.a = 10;
cloneObj.b.c = 20;

console.log(obj.a);     // 1
console.log(obj.b.c);   // 20

2.3 Array.prototype.slice()

const arr = [1, 2, { a: 3 }];
const cloneArr = arr.slice();

cloneArr[0] = 10;
cloneArr[2].a = 30;

console.log(arr[0]);     // 1
console.log(arr[2].a);   // 30

2.4 Array.prototype.concat()

const arr = [1, 2, { a: 3 }];
const cloneArr = arr.concat();

cloneArr[0] = 10;
cloneArr[2].a = 30;

console.log(arr[0]);     // 1
console.log(arr[2].a);   // 30

浅拷贝的局限性

❌ 无法处理嵌套对象
❌ 无法处理循环引用
❌ 只适用于简单的数据结构

三、深拷贝方法对比

3.1 JSON.parse(JSON.stringify())

实现方式：

const obj = {
  name: 'Alice',
  age: 18,
  hobbies: ['reading', 'coding'],
  info: { city: 'Beijing' }
};

const cloneObj = JSON.parse(JSON.stringify(obj));
cloneObj.name = 'Bob';
cloneObj.info.city = 'Shanghai';

console.log(obj.name);         // 'Alice'
console.log(obj.info.city);    // 'Beijing'

优点：

  • ✅ 代码简单，一行搞定
  • ✅ 性能较好
  • ✅ 浏览器原生支持

缺点：

  • ❌ 无法处理 undefined（会被忽略）
  • ❌ 无法处理 Symbol（会被忽略）
  • ❌ 无法处理 Function（会被忽略）
  • ❌ 无法处理循环引用（报错）
  • ❌ 无法处理 Date（转为字符串）
  • ❌ 无法处理 RegExp（转为空对象）
  • ❌ 无法处理 Map、Set（转为空对象）
  • ❌ 丢失原型链

测试案例：

const obj = {
  a: undefined,
  b: Symbol('test'),
  c: function() { console.log('hello'); },
  d: new Date(),
  e: /abc/gi,
  f: new Map([['key', 'value']]),
  g: new Set([1, 2, 3]),
  h: NaN,
  i: Infinity
};

const cloneObj = JSON.parse(JSON.stringify(obj));
console.log(cloneObj);
// {
//   d: "2023-12-23T00:00:00.000Z",  // Date 变成字符串
//   e: {},                           // RegExp 变成空对象
//   f: {},                           // Map 变成空对象
//   g: {},                           // Set 变成空对象
//   h: null,                         // NaN 变成 null
//   i: null                          // Infinity 变成 null
// }
// undefined、Symbol、Function 都被忽略了！

3.2 lodash.cloneDeep

实现方式：

import _ from 'lodash';

const obj = {
  name: 'Alice',
  info: { age: 18 },
  date: new Date(),
  reg: /test/gi,
  fn: () => console.log('hello')
};

const cloneObj = _.cloneDeep(obj);
console.log(cloneObj);

优点：

  • ✅ 功能最全面
  • ✅ 性能优化到位
  • ✅ 支持所有数据类型
  • ✅ 处理循环引用
  • ✅ 保留原型链

缺点：

  • ❌ 需要引入第三方库（约 70KB gzipped）
  • ❌ 对于简单场景过于重量级

3.3 structuredClone（现代浏览器）

实现方式：

const obj = {
  name: 'Alice',
  info: { age: 18 },
  date: new Date(),
  map: new Map([['key', 'value']]),
  set: new Set([1, 2, 3])
};

const cloneObj = structuredClone(obj);
console.log(cloneObj);

优点：

  • ✅ 浏览器原生支持
  • ✅ 支持 Date、Map、Set、RegExp、ArrayBuffer 等
  • ✅ 支持循环引用
  • ✅ 性能优秀

缺点：

  • ❌ 无法处理 Function（报错）
  • ❌ 无法处理 Error（报错）
  • ❌ 无法处理 DOM 节点（报错）
  • ❌ 兼容性：Chrome 98+、Firefox 94+、Safari 15.4+

3.5 手写递归实现（基础版）

function deepClone(obj, hash = new WeakMap()) {
  // 1. null 或 undefined
  if (obj === null || obj === undefined) return obj;
  
  // 2. 原始类型直接返回
  if (typeof obj !== 'object') return obj;
  
  // 3. 处理循环引用
  if (hash.has(obj)) return hash.get(obj);
  
  // 4. 特殊对象类型
  if (obj instanceof Date) return new Date(obj);
  if (obj instanceof RegExp) return new RegExp(obj);
  
  // 5. 创建新对象
  const cloneObj = Array.isArray(obj) ? [] : {};
  hash.set(obj, cloneObj);
  
  // 6. 递归复制
  for (let key in obj) {
    if (obj.hasOwnProperty(key)) {
      cloneObj[key] = deepClone(obj[key], hash);
    }
  }
  
  return cloneObj;
}

// 测试
const obj = { name: 'Alice', info: { age: 18 } };
obj.self = obj; // 循环引用
const cloneObj = deepClone(obj);
console.log(cloneObj);

优点：

  • ✅ 支持循环引用
  • ✅ 支持 Date、RegExp
  • ✅ 代码相对简单

缺点：

  • ❌ 无法处理 Map、Set
  • ❌ 无法处理 Symbol 属性
  • ❌ 无法处理 Function
  • ❌ 丢失原型链

四、终极深拷贝实现（完整版）

下面是支持所有数据类型的完整实现：

// 类型常量定义
const mapTag = "[object Map]";
const setTag = "[object Set]";
const arrayTag = "[object Array]";
const objectTag = "[object Object]";
const argsTag = "[object Arguments]";
const boolTag = "[object Boolean]";
const dateTag = "[object Date]";
const numberTag = "[object Number]";
const stringTag = "[object String]";
const symbolTag = "[object Symbol]";
const errorTag = "[object Error]";
const regexpTag = "[object RegExp]";
const funcTag = "[object Function]";

const deepTag = [mapTag, setTag, arrayTag, objectTag, argsTag];

// 获取数据类型
function getType(target) {
  return Object.prototype.toString.call(target);
}

// 判断是否为引用类型
function isObject(target) {
  const type = typeof target;
  return target !== null && (type === "object" || type === "function");
}

// 初始化对象原型
function initPrototype(target) {
  const Ctor = target.constructor;
  return new Ctor();
}

// 克隆正则表达式
function cloneReg(target) {
  const reFlags = /\w*$/;
  const result = new target.constructor(target.source, reFlags.exec(target));
  result.lastIndex = target.lastIndex;
  return result;
}

// 克隆 Symbol
function cloneSymbol(target) {
  return Object(Symbol.prototype.valueOf.call(target));
}

// 克隆函数
function cloneFunction(func) {
  const bodyReg = /(?<={)(.|\n)+(?=})/m;
  const paramReg = /(?<=\().+(?=\)\s+{)/;
  const funcString = func.toString();
  
  // 判断是否是箭头函数（箭头函数没有原型）
  if (func.prototype) {
    const param = paramReg.exec(funcString);
    const body = bodyReg.exec(funcString);
    if (body) {
      if (param) {
        const paramArr = param[0].split(",");
        return new Function(...paramArr, body[0]);
      } else {
        return new Function(body[0]);
      }
    } else {
      return func; // 无法解析，返回原函数
    }
  } else {
    // 箭头函数使用 eval
    return eval(funcString);
  }
}

// 克隆其他特殊类型
function cloneOtherType(target, type) {
  const Ctor = target.constructor;
  switch (type) {
    case boolTag:
    case numberTag:
    case stringTag:
    case errorTag:
    case dateTag:
      return new Ctor(target);
    case regexpTag:
      return cloneReg(target);
    case symbolTag:
      return cloneSymbol(target);
    case funcTag:
      return cloneFunction(target);
    default:
      return null;
  }
}

// 遍历数组或对象
function forEach(array, callback) {
  let index = -1;
  const length = array.length;
  while (++index < length) {
    callback(array[index], index);
  }
  return array;
}

// 主函数：深度克隆
function clone(target, map = new WeakMap()) {
  // 1. 原始数据类型直接返回
  if (!isObject(target)) {
    return target;
  }
  
  // 2. 获取数据类型
  const type = getType(target);
  let cloneObj = null;
  
  // 3. 判断是否需要深度克隆的类型
  if (deepTag.includes(type)) {
    cloneObj = initPrototype(target);
  } else {
    return cloneOtherType(target, type);
  }
  
  // 4. 处理循环引用
  if (map.get(target)) {
    return map.get(target);
  }
  map.set(target, cloneObj);
  
  // 5. 克隆 Set
  if (type === setTag) {
    target.forEach((value) => {
      cloneObj.add(clone(value, map));
    });
    return cloneObj;
  }
  
  // 6. 克隆 Map
  if (type === mapTag) {
    target.forEach((value, key) => {
      cloneObj.set(key, clone(value, map));
    });
    return cloneObj;
  }
  
  // 7. 克隆对象和数组
  const keys = type === arrayTag ? undefined : Object.keys(target);
  forEach(keys || target, (value, key) => {
    if (keys) {
      key = value;
    }
    cloneObj[key] = clone(target[key], map);
  });
  
  // 8. 克隆 Symbol 属性
  const symbolKeys = type === objectTag ? Object.getOwnPropertySymbols(target) : undefined;
  if (symbolKeys) {
    forEach(symbolKeys, (symbolKey) => {
      cloneObj[symbolKey] = clone(target[symbolKey], map);
    });
  }
  
  return cloneObj;
}

// 测试用例
const sym = Symbol('test');
const obj = {
  [sym]: { value: 'symbol value' },
  name: 'Alice',
  age: 18,
  hobbies: ['reading', 'coding'],
  info: {
    city: 'Beijing',
    scores: [90, 95, 88]
  },
  date: new Date(),
  reg: /test/gi,
  map: new Map([['key1', 'value1'], ['key2', 'value2']]),
  set: new Set([1, 2, 3]),
  fn: function(a, b) {
    return a + b;
  },
  arrowFn: (x) => x * 2
};

// 添加循环引用
obj.self = obj;

const cloneObj = clone(obj);
console.log('原对象:', obj);
console.log('克隆对象:', cloneObj);
console.log('循环引用测试:', cloneObj.self === cloneObj); // true
console.log('Symbol 属性测试:', cloneObj[sym]); // { value: 'symbol value' }
console.log('函数测试:', cloneObj.fn(1, 2)); // 3
console.log('箭头函数测试:', cloneObj.arrowFn(5)); // 10

六、方法选择指南

6.1 决策树

需要深拷贝？
  ├─ 数据是否包含特殊类型（Date、RegExp、Map、Set、Function）？
  │   ├─ 否 → 使用 JSON.parse(JSON.stringify(obj))
  │   └─ 是 → 是否需要处理循环引用？
  │            ├─ 否 → 使用递归浅拷贝
  │            └─ 是 → 是否兼容旧浏览器？
  │                      ├─ 是 → 手写完整版或使用 lodash
  │                      └─ 否 → 使用 structuredClone
  └─ 是否在意包体积？
      ├─ 否 → 使用 lodash.cloneDeep
      └─ 是 → 手写实现或使用 structuredClone

6.2 推荐方案

场景 推荐方法 理由
简单对象（无特殊类型） JSON.parse(JSON.stringify()) 简单、性能好
现代浏览器环境 structuredClone() 原生支持、功能完善
生产环境（已有 lodash） _.cloneDeep() 功能最全、经过充分测试
面试/学习 手写完整版 展示对细节的掌握
兼容旧浏览器 手写完整版 可控性强

七、常见问题与陷阱

7.1 循环引用问题

// 问题：JSON 方法会报错
const obj = { name: 'Alice' };
obj.self = obj;
JSON.parse(JSON.stringify(obj)); // TypeError: Converting circular structure to JSON

// 解决：使用 WeakMap 缓存
function deepClone(obj, hash = new WeakMap()) {
  if (hash.has(obj)) return hash.get(obj);
  // ... 其余代码
  hash.set(obj, cloneObj);
  // ... 其余代码
}

7.2 原型链丢失

// 问题：JSON 方法丢失原型
function Person(name) {
  this.name = name;
}
Person.prototype.sayHi = function() {
  console.log(`Hi, I'm ${this.name}`);
};

const alice = new Person('Alice');
const cloneAlice = JSON.parse(JSON.stringify(alice));
console.log(cloneAlice instanceof Person); // false
cloneAlice.sayHi(); // TypeError: cloneAlice.sayHi is not a function

// 解决：使用 constructor
function clone(target) {
  const cloneObj = new target.constructor();
  // ... 复制属性
  return cloneObj;
}

7.3 函数克隆的局限性

// 问题：函数克隆无法保留闭包
function createCounter() {
  let count = 0;
  return function() {
    return ++count;
  };
}

const counter = createCounter();
counter(); // 1
counter(); // 2

const cloneCounter = cloneFunction(counter);
cloneCounter(); // 1 - 闭包状态丢失！

结论：函数克隆只能克隆函数结构和代码，无法克隆闭包中的状态。

8.2 最佳实践

  1. 日常开发：优先使用 structuredClone()，不支持时降级到 JSON.parse
  2. 生产环境：使用 lodash.cloneDeep，功能完善且经过充分测试
  3. 性能敏感：使用 JSON.parse 或缓存克隆结果
  4. 学习/面试：理解手写实现的每个细节，包括循环引用、Symbol 属性、原型链等

8.3 面试要点

Q1：深拷贝和浅拷贝的区别？
A：深拷贝递归复制所有层级，新旧对象完全独立；浅拷贝只复制第一层，嵌套对象仍共享引用。

Q2：JSON.parse(JSON.stringify()) 的缺点？
A：无法处理 undefined、Symbol、Function、循环引用，Date 会变成字符串，RegExp 会变成空对象。

Q3：如何处理循环引用？
A：使用 WeakMap 缓存已克隆的对象，遇到已存在的对象直接返回缓存值。

Q4：手写深拷贝需要考虑哪些情况？
A：数据类型判断、循环引用、Symbol 属性、原型链、特殊对象（Date、RegExp、Map、Set）、函数克隆。
