JavaScript事件循环：深入理解宏任务与微任务

发表于2024|更新于2026|前端基础
|字数总计:2.7k|阅读时长:11分钟

前言

事件循环（Event Loop）是 JavaScript 异步编程的核心机制，也是面试中的高频考点。理解事件循环，才能真正理解 JavaScript 的异步执行顺序。本文将深入剖析事件循环的工作原理，帮你彻底搞懂宏任务与微任务的执行机制。

一、为什么需要事件循环？

1.1 JavaScript 的单线程特性

JavaScript 从诞生之初就是一门单线程语言，这意味着它同一时间只能做一件事。

// 单线程：代码按顺序执行
console.log(1);
console.log(2);
console.log(3);
// 输出：1 2 3（严格按顺序）

为什么是单线程？

JavaScript 的主要用途是处理网页交互和 DOM 操作。如果是多线程，两个线程同时操作一个 DOM 节点，一个修改、一个删除，就会产生冲突。为了避免复杂的线程同步问题，JavaScript 选择了单线程。

1.2 单线程的问题

单线程意味着所有任务都要排队执行，如果前一个任务耗时很长，后面的任务就要一直等待：

// 同步阻塞问题
console.log('开始');

// 模拟耗时操作（同步）
const start = Date.now();
while (Date.now() - start < 3000) {} // 阻塞3秒

console.log('结束');
// 用户在这3秒内无法进行任何操作！

1.3 解决方案：异步 + 事件循环

为了解决单线程的阻塞问题，JavaScript 将任务分为两类：

类型 特点 示例
同步任务 立即执行，在主线程上排队 变量赋值、函数调用
异步任务 不进入主线程，等待结果后再执行 setTimeout、AJAX、Promise

事件循环就是协调同步任务和异步任务执行顺序的机制。

二、核心概念

2.1 调用栈（Call Stack）

调用栈是一个后进先出（LIFO）的结构，用于记录代码执行的位置：

function multiply(a, b) {
  return a * b;
}

function square(n) {
  return multiply(n, n);
}

function printSquare(n) {
  const result = square(n);
  console.log(result);
}

printSquare(4);

调用栈执行过程：

1. printSquare(4) 入栈
2. square(4) 入栈
3. multiply(4, 4) 入栈
4. multiply 返回 16，出栈
5. square 返回 16，出栈
6. console.log(16) 入栈
7. console.log 执行完毕，出栈
8. printSquare 执行完毕，出栈

![调用栈示意图]

2.2 任务队列（Task Queue）

任务队列是一个先进先出（FIFO）的结构，用于存放等待执行的异步任务回调。

JavaScript 中有两种任务队列：

┌─────────────────────────────────────┐
│           任务队列                   │
├─────────────────┬───────────────────┤
│   宏任务队列     │    微任务队列      │
│ (Macrotask)     │   (Microtask)     │
├─────────────────┼───────────────────┤
│ setTimeout      │ Promise.then      │
│ setInterval     │ Promise.catch     │
│ setImmediate    │ Promise.finally   │
│ I/O             │ process.nextTick  │
│ UI Rendering    │ MutationObserver  │
│ requestAnimation│ queueMicrotask    │
└─────────────────┴───────────────────┘

2.3 宏任务与微任务

宏任务（Macrotask）

API 说明
setTimeout 延迟执行
setInterval 定时执行
setImmediate Node.js 环境，立即执行
I/O 文件读写、网络请求
UI Rendering 浏览器渲染
requestAnimationFrame 动画帧回调

微任务（Microtask）

API 说明
Promise.then/catch/finally Promise 回调
process.nextTick Node.js 环境，优先级最高
MutationObserver DOM 变化监听
queueMicrotask 手动添加微任务

关键区别：

// 宏任务
setTimeout(() => console.log('setTimeout'), 0);

// 微任务
Promise.resolve().then(() => console.log('Promise'));

console.log('同步代码');

// 输出顺序：同步代码 → Promise → setTimeout

2.4 任务优先级

浏览器环境

同步代码 > 微任务 > 宏任务

微任务内部优先级：
Promise.then/catch/finally > MutationObserver > queueMicrotask

// 验证微任务优先级
MutationObserver = window.MutationObserver || function() {};
queueMicrotask(() => console.log('queueMicrotask'));
Promise.resolve().then(() => console.log('Promise'));
// 输出：Promise → queueMicrotask（Promise 更早加入队列）

Node.js 环境

同步代码 > process.nextTick > Promise（微任务） > 宏任务

// Node.js 环境
process.nextTick(() => console.log('nextTick'));
Promise.resolve().then(() => console.log('Promise'));
setTimeout(() => console.log('setTimeout'), 0);

// 输出：nextTick → Promise → setTimeout

Node.js 完整优先级：

process.nextTick > Promise微任务 > timer(setTimeout/setInterval) 
> pending callbacks > idle/prepare > poll 
> check(setImmediate) > close callbacks

三、事件循环执行流程

3.1 完整执行机制

┌───────────────────────────────────────────┐
│                  事件循环                   │
│                                           │
│  ┌─────────────────────────────────────┐  │
│  │  1. 从调用栈执行同步代码              │  │
│  │     直到调用栈为空                    │  │
│  └──────────────────┬──────────────────┘  │
│                     ▼                     │
│  ┌─────────────────────────────────────┐  │
│  │  2. 检查微任务队列                    │  │
│  │     - 如果有，依次执行所有微任务       │  │
│  │     - 执行完一个，继续检查下一个       │  │
│  │     - 直到微任务队列为空              │  │
│  └──────────────────┬──────────────────┘  │
│                     ▼                     │
│  ┌─────────────────────────────────────┐  │
│  │  3. UI 渲染（如果有需要）             │  │
│  └──────────────────┬──────────────────┘  │
│                     ▼                     │
│  ┌─────────────────────────────────────┐  │
│  │  4. 从宏任务队列取出一个任务执行       │  │
│  └──────────────────┬──────────────────┘  │
│                     │                     │
│                     ▼                     │
│         ┌─────────────────────┐          │
│         │  回到步骤 2         │          │
│         │  （新一轮循环）      │          │
│         └─────────────────────┘          │
└───────────────────────────────────────────┘

3.2 执行流程口诀

同步先执行，
栈空查微任，
微任全清完，
宏任取一个，
循环往复行。

3.3 详细步骤

  1. 执行同步代码：将同步任务压入调用栈执行，直到调用栈为空
  2. 检查微任务队列：依次执行所有微任务，直到队列为空
  3. UI 渲染：浏览器会检查是否需要重新渲染页面
  4. 执行一个宏任务：从宏任务队列取出第一个任务执行
  5. 回到步骤 2：开始新一轮的事件循环

注意： 每次执行完一个宏任务，都会检查并执行所有微任务！

四、经典面试题解析

4.1 基础题目

console.log(1);

setTimeout(() => {
  console.log(2);
}, 0);

Promise.resolve().then(() => {
  console.log(3);
});

console.log(4);

执行过程分析：

步骤 操作 调用栈 微任务队列 宏任务队列 输出
1 console.log(1) [log] [] [] 1
2 setTimeout [setTimeout] [] [callback] -
3 Promise.then [then] [callback] [setTimeout回调] -
4 console.log(4) [log] [Promise回调] [setTimeout回调] 4
5 同步结束 [] [Promise回调] [setTimeout回调] -
6 执行微任务 [] [] [setTimeout回调] 3
7 执行宏任务 [] [] [] 2

输出结果：1 4 3 2

4.2 进阶题目

console.log('script start');

async function async1() {
  await async2();
  console.log('async1 end');
}

async function async2() {
  console.log('async2 end');
}

async1();

setTimeout(function () {
  console.log('setTimeout');
}, 0);

new Promise((resolve) => {
  console.log('Promise');
  resolve();
}).then(function () {
  console.log('promise1');
}).then(function () {
  console.log('promise2');
});

console.log('script end');

关键点：await 后面的代码相当于 Promise.then

// async1 等价于
function async1() {
  return Promise.resolve(async2()).then(() => {
    console.log('async1 end');
  });
}

执行过程：

步骤 操作 输出
1 console.log('script start') script start
2 调用 async1()，进入 async2() async2 end
3 await 后续代码加入微任务 -
4 setTimeout 加入宏任务队列 -
5 new Promise 执行同步部分 Promise
6 .then 加入微任务队列 -
7 console.log('script end') script end
8 执行微任务：async1 end async1 end
9 执行微任务：promise1 promise1
10 .then 链式调用，promise2 加入微任务 -
11 执行微任务：promise2 promise2
12 执行宏任务：setTimeout setTimeout

输出结果：

script start
async2 end
Promise
script end
async1 end
promise1
promise2
setTimeout

4.3 复杂嵌套题目

new Promise((resolve, reject) => {
  console.log('Promise1');
  resolve();
})
  .then(() => {
    console.log('Promise2');
    new Promise((resolve, reject) => {
      console.log('Promise3');
      resolve();
    })
      .then(() => {
        console.log('Promise4');
      })
      .then(() => {
        console.log('Promise5');
      });
  })
  .then(() => {
    console.log('Promise6');
  });

setTimeout(() => {
  console.log('setTimeout');
}, 0);

console.log('script end');

解析要点：

  1. Promise 构造函数中的代码是同步执行的
  2. 第一个 .then 的回调执行时，内部的 Promise 又产生新的微任务
  3. 注意 Promise 链式调用的时序

输出结果：

Promise1
script end
Promise2
Promise3
Promise4
Promise6
Promise5
setTimeout

详细分析：

// 第一次微任务检查
微任务队列: [then1回调]
执行 then1 回调:
  - 输出 Promise2
  - 创建新 Promise（同步执行 console.log('Promise3')）
  - 新 Promise 的 then 加入微任务队列
  - 外层第二个 then 加入微任务队列（等待第一个 then 返回的 Promise）

// 第二次微任务检查
微任务队列: [内层then回调, 外层then2回调]
执行内层 then 回调:
  - 输出 Promise4
  - 链式 then5 加入微任务队列
执行外层 then2 回调:
  - 输出 Promise6

// 第三次微任务检查
微任务队列: [then5回调]
执行 then5 回调:
  - 输出 Promise5

// 执行宏任务
输出 setTimeout

4.4 async/await 与 Promise 的转换

async function async1() {
  console.log('async1 start');
  await async2();
  console.log('async1 end');
}

async function async2() {
  console.log('async2');
}

async1();

console.log('script end');

转换理解：

function async1() {
  console.log('async1 start');
  return Promise.resolve(async2()).then(() => {
    console.log('async1 end');
  });
}

输出：

async1 start
async2
script end
async1 end

五、Node.js 事件循环

Node.js 的事件循环与浏览器有所区别，基于 libuv 实现。不同的node版本优先级也有细微的差异。

5.1 Node.js 事件循环阶段

   ┌───────────────────────┐
┌─>│        timers         │ setTimeout/setInterval
│  └───────────┬───────────┘
│  ┌───────────┴───────────┐
│  │     pending callbacks  │ I/O 回调
│  └───────────┬───────────┘
│  ┌───────────┴───────────┐
│  │       idle, prepare   │ 内部使用
│  └───────────┬───────────┘
│  ┌───────────┴───────────┐
│  │        poll           │ 轮询，等待新 I/O 事件
│  └───────────┬───────────┘
│  ┌───────────┴───────────┐
│  │        check          │ setImmediate
│  └───────────┬───────────┘
│  ┌───────────┴───────────┐
└──┤    close callbacks    │ close 事件
   └───────────────────────┘

5.2 process.nextTick

Node.js 中 process.nextTick 的优先级高于 Promise：

// Node.js 环境
process.nextTick(() => {
  console.log('nextTick');
});

Promise.resolve().then(() => {
  console.log('Promise');
});

// 输出：nextTick → Promise

5.3 setTimeout vs setImmediate

在 Node.js 中，setTimeout 和 setImmediate 的执行顺序取决于上下文：

// 在 I/O 回调中
const fs = require('fs');

fs.readFile(__filename, () => {
  setTimeout(() => console.log('timeout'), 0);
  setImmediate(() => console.log('immediate'));
});
// 输出：immediate → timeout（setImmediate 先执行）

// 在主模块中
setTimeout(() => console.log('timeout'), 0);
setImmediate(() => console.log('immediate'));
// 输出顺序不确定（取决于性能）

六、实战技巧与最佳实践

6.1 合理使用微任务

// ❌ 不推荐：大量微任务可能阻塞 UI 渲染
function processArray(arr) {
  arr.forEach(item => {
    Promise.resolve().then(() => {
      // 处理每个元素
    });
  });
}

// ✅ 推荐：使用宏任务让 UI 有机会渲染
function processArray(arr) {
  let i = 0;
  function process() {
    if (i < arr.length) {
      // 处理 arr[i]
      i++;
      setTimeout(process, 0); // 让出主线程
    }
  }
  process();
}

6.2 避免任务队列饥饿

// ❌ 微任务不断添加，宏任务无法执行
function infiniteMicrotasks() {
  Promise.resolve().then(() => {
    console.log('微任务');
    infiniteMicrotasks();
  });
}
infiniteMicrotasks();
setTimeout(() => console.log('宏任务'), 0);
// "宏任务" 永远不会执行！

// ✅ 使用宏任务让出控制权
function betterApproach() {
  Promise.resolve().then(() => {
    console.log('微任务');
    setTimeout(betterApproach, 0);
  });
}

6.3 利用 queueMicrotask

// 使用 queueMicrotask 添加微任务
function doSomething() {
  queueMicrotask(() => {
    console.log('微任务执行');
  });
}

// 在自定义类中使用
class EventEmitter {
  constructor() {
    this.listeners = {};
  }
  
  emit(event, data) {
    // 使用微任务确保异步通知
    queueMicrotask(() => {
      if (this.listeners[event]) {
        this.listeners[event].forEach(fn => fn(data));
      }
    });
  }
}

七、总结

7.1 核心要点

1. JavaScript 是单线程语言
2. 同步代码在调用栈中执行
3. 异步任务分为宏任务和微任务
4. 事件循环流程：同步 → 微任务 → 宏任务 → 循环
5. 每次宏任务执行后，都会清空微任务队列

7.2 执行顺序速记

同步代码 → process.nextTick(Node.js) → 微任务 → 宏任务

7.3 面试必记

问题 答案
为什么 JS 是单线程？ 避免 DOM 操作冲突
微任务优先级为什么高于宏任务？ 确保异步操作结果尽快处理
setTimeout(fn, 0) 什么时候执行？ 下一次事件循环的宏任务阶段
await 后面的代码什么时候执行？ 作为微任务执行

理解事件循环，是掌握 JavaScript 异步编程的关键！
