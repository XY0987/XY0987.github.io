CommonJS和ES Module的区别

发表于2024-02-03|更新于2026-03-14|前端基础
|字数总计:856|阅读时长:3分钟

概述

在最开始写 js 时，并没有模块的概念，一般都是采用 script 的模式去引入，但是随着项目的越来越大，script 引入的方式显得非常复杂。在 ES6 之前，模块化的方案主要有 CommonJS 和 AMD 两种，前者用于服务器（require 语法就是），后者用于浏览器。在 ES6 之后，社区制定了 ES Module（import），由于 import 时静态的，在之后又推出了 import()函数，用来实现模块化。 本文主要是总结两种模块化的区别，具体用法可以看ES6 文档

二者区别

引入方式

CommonJS 是动态导入的，可以在任何地方引入，比如：

if(true){
  const a =require('./a.js)
  a()
}

而 ES Module 是静态导入，在编译时运行，所以导入只能写在最顶层作用域中，同时 import 具有提升效果，会提升到整个模块的顶部，不过在之后的版本中，提供了 import()函数，可以让我们动态导入

a()//可以执行，具有提升效果
import a from './a.js'

//报错
if(true){
  import a from './a.js
    a()
}

//使用import()函数实现动态导入
if(true){
  import('./a.js)
    .then(a=>{
      a()
    })
    }

import()有点类似于 require()方法，区别主要是前者是异步加载，后者是同步加载，import()返回的是一个 Promise 对象

使用语法

CommonJS

// 导出
const obj = {
    name: 'hello world'
}
module.exports = obj

// 导入
const obj = require('./a.js')
console.log(obj.name)

CommonJS 输出的是一个值拷贝（实际上是给变量赋值，如果是基本数据类型的话，导出值变化，不会影响导入值的变化）

// b.js
let a = 1
let obj = {
    name: '原始值'
}

setTimeout(() => {
    a = 100
    obj.name = '修改值'
    console.log('值改变了', a, obj);//100  {name:'修改值'}
}, 300)

module.exports = {
    a,
    obj
}


// a.js
let a = require('./b')
setTimeout(() => {
    console.log(a.a);//1
    console.log(a.obj);//{name:'修改值'}
}, 2000)

ES Module

// 导出
export const title = 'es module'
export const message = 'hello world'
const obj = {
    name: 'hello world'
}
export default obj

// 导入
import obj, { title, message } from 'a.js'
console.log(title, message, obj.name)

ES Module 模块是值引用并且他导出的值是只读的（如果是引用数据类型，我们可以修改内部属性的值）

// 导出
export let title = 'hello hi'
const obj = {
    name: 'tom'
}
export default obj
setTimeout(() => {
    title = 'hello world'
    obj.name = 'tony'
}, 2000)

// 导入
import obj, { title } from './a.js'
console.log(title)
console.log(obj.name)
setTimeout(() => {
    console.log(title)
    console.log(obj.name)
}, 4000)

/*hello hi
tom
hello world
tony
*/

我们可以看出来，即使是基本数据类型，导出的值改变了，也会影响导入的值

加载运行和加载模式

CommonJS 是在运行是加载，因为 CommonJS 是导出整个对象，需要在脚本运行完成后才能生成 ESModule 是编译时加载，它是在静态分析编译输出接口，它的对外接口只是一种静态的定义

同时需要注意一点的是，CommonJS 由于是在服务器端的规范，读取资源非常的快，所以采用的是同步的方式加载，而 ESModule 是浏览器请求网络资源，为避免卡顿，所以使用的异步的方式加载
