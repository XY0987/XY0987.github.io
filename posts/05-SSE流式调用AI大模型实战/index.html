SSE流式调用AI大模型实战

发表于2025|更新于2026|AI相关
|字数总计:2.2k|阅读时长:10分钟

前言：SSE（Server-Sent Events）流式输出让回答逐字出现，这是 AI 应用从”能用”到”好用”的关键一步。

SSE 不是什么新技术，但 AI 让它焕发了第二春。在 AI 应用场景下，SSE 是性价比最高的传输方案——比 WebSocket 简单，比轮询高效，完美匹配”服务器单向推送”的场景。

为什么必须要流式输出

对比一下体验差异：

// ❌ 非流式：用户等待 8-15 秒，期间没有任何反馈
const response = await fetch('/api/chat', {
  method: 'POST',
  body: JSON.stringify({ message: '写一篇2000字的文章' })
});
const data = await response.json();
// 用户心理："卡了？崩了？要不要刷新？"

// ✅ 流式：200ms 内就看到第一个字，逐字出现
// 用户心理："它在思考，在写了，不错不错"

SSE 协议基础

SSE 数据格式

SSE 有严格的格式规范，每条消息由字段名和值组成，消息之间用空行分隔：

data: {"content": "你"}

data: {"content": "好"}

data: {"content": "世"}

data: {"content": "界"}

data: [DONE]

支持的字段：

  • data: —— 数据内容（最常用）
  • event: —— 自定义事件类型
  • id: —— 消息ID（用于断线重连）
  • retry: —— 重连间隔（毫秒）

SSE vs WebSocket vs 长轮询

特性 SSE WebSocket 长轮询
方向 单向（服务器→客户端） 双向 单向
协议 HTTP WS HTTP
复杂度 ⭐低 ⭐⭐中 ⭐低
自动重连 ✅内置 ❌手动实现 ❌手动实现
部署难度 低（标准HTTP） 中（需WS支持） 低
AI场景适配 ⭐⭐⭐完美 ⭐⭐过度设计 ⭐低效

AI 流式输出场景 → SSE，AI 回答是典型的”服务器单向推送”，不需要 WebSocket 的双向通信复杂性。

实战：完整的流式聊天应用

服务端（Node.js + Express）

// server.mjs —— SSE流式服务端
import express from 'express';
import OpenAI from 'openai';
import cors from 'cors';

const app = express();
app.use(cors());
app.use(express.json());

const openai = new OpenAI({ apiKey: process.env.OPENAI_API_KEY });

app.post('/api/chat/stream', async (req, res) => {
  const { messages } = req.body;

  // ① 设置SSE响应头 —— 这几个缺一不可
  res.setHeader('Content-Type', 'text/event-stream');
  res.setHeader('Cache-Control', 'no-cache');
  res.setHeader('Connection', 'keep-alive');
  // 禁用 Nginx 缓冲（避免nginx缓冲机制导致接口一直无响应）
  res.setHeader('X-Accel-Buffering', 'no');

  // ② 处理客户端断开连接
  let isClosed = false;
  req.on('close', () => {
    isClosed = true;
  });

  try {
    // ③ 调用 OpenAI 流式API
    const stream = await openai.chat.completions.create({
      model: 'gpt-4',
      messages,
      stream: true
    });

    // ④ 心跳保活（防止代理服务器超时断开）
    const heartbeat = setInterval(() => {
      if (!isClosed) {
        res.write(': heartbeat');  // 冒号开头是SSE注释，不影响客户端
      }
    }, 15000);

    // ⑤ 逐块转发
    for await (const chunk of stream) {
      if (isClosed) break;

      const content = chunk.choices[0]?.delta?.content || '';
      const finishReason = chunk.choices[0]?.finish_reason;

      if (content) {
        res.write(`data: ${JSON.stringify({ type: 'content', content })}`);
      }

      if (finishReason === 'stop') {
        res.write(`data: ${JSON.stringify({ type: 'done' })}`);
      }
    }

    clearInterval(heartbeat);
    res.end();

  } catch (error) {
    console.error('Stream error:', error);
    if (!isClosed) {
      res.write(`data: ${JSON.stringify({ type: 'error', message: error.message })}\n\n`);
      res.end();
    }
  }
});

app.listen(3000, () => console.log('🚀 Server running on http://localhost:3000'));

客户端（通用 fetch + ReadableStream）

// streamClient.mjs —— 通用流式客户端
// 使用 fetch + ReadableStream（推荐，支持POST、自定义header）

async function* streamChat(messages) {
  const response = await fetch('http://localhost:3000/api/chat/stream', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${token}`  // 支持自定义header
    },
    body: JSON.stringify({ messages })
  });

  if (!response.ok) {
    throw new Error(`HTTP ${response.status}: ${response.statusText}`);
  }

  const reader = response.body.getReader();
  const decoder = new TextDecoder();
  let buffer = '';

  while (true) {
    const { done, value } = await reader.read();
    if (done) break;

    buffer += decoder.decode(value, { stream: true });

    // 按行分割处理SSE格式
    const lines = buffer.split('\n');
    buffer = lines.pop() || '';  // 保留最后一个不完整的行

    for (const line of lines) {
      if (line.startsWith('data: ')) {
        const data = line.slice(6).trim();
        if (data === '[DONE]') return;

        try {
          const parsed = JSON.parse(data);
          if (parsed.type === 'content') {
            yield parsed.content;  // 逐字产出
          } else if (parsed.type === 'error') {
            throw new Error(parsed.message);
          } else if (parsed.type === 'done') {
            return;
          }
        } catch (e) {
          if (e instanceof SyntaxError) continue;  // 跳过非JSON行
          throw e;
        }
      }
    }
  }
}

// 使用方式
for await (const chunk of streamChat([{ role: 'user', content: '你好' }])) {
  process.stdout.write(chunk);
}

前端 React 组件

// useStreamChat.js —— React Hook
import { useState, useCallback, useRef } from 'react';

export function useStreamChat() {
  const [messages, setMessages] = useState([]);
  const [currentContent, setCurrentContent] = useState('');
  const [isStreaming, setIsStreaming] = useState(false);
  const abortRef = useRef(null);

  const sendMessage = useCallback(async (content) => {
    const userMsg = { role: 'user', content };
    const newMessages = [...messages, userMsg];
    setMessages(newMessages);
    setIsStreaming(true);
    setCurrentContent('');

    abortRef.current = new AbortController();

    try {
      const res = await fetch('/api/chat/stream', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ messages: newMessages }),
        signal: abortRef.current.signal
      });

      const reader = res.body.getReader();
      const decoder = new TextDecoder();
      let buffer = '';
      let fullContent = '';

      while (true) {
        const { done, value } = await reader.read();
        if (done) break;

        buffer += decoder.decode(value, { stream: true });
        const lines = buffer.split('\n');
        buffer = lines.pop() || '';

        for (const line of lines) {
          if (line.startsWith('data: ')) {
            try {
              const data = JSON.parse(line.slice(6));
              if (data.type === 'content') {
                fullContent += data.content;
                setCurrentContent(fullContent);
              } else if (data.type === 'done') {
                setMessages(prev => [...prev, { role: 'assistant', content: fullContent }]);
                setCurrentContent('');
              }
            } catch {}
          }
        }
      }
    } catch (error) {
      if (error.name !== 'AbortError') {
        console.error('Stream error:', error);
      }
    } finally {
      setIsStreaming(false);
    }
  }, [messages]);

  const stopStreaming = useCallback(() => {
    abortRef.current?.abort();
  }, []);

  return { messages, currentContent, isStreaming, sendMessage, stopStreaming };
}

// ChatApp.jsx —— 聊天界面
import { useState } from 'react';
import { useStreamChat } from './useStreamChat';

export function ChatApp() {
  const { messages, currentContent, isStreaming, sendMessage, stopStreaming } = useStreamChat();
  const [input, setInput] = useState('');

  const handleSubmit = (e) => {
    e.preventDefault();
    if (!input.trim() || isStreaming) return;
    sendMessage(input.trim());
    setInput('');
  };

  return (
    <div style={{ maxWidth: 700, margin: '0 auto', padding: 20 }}>
      <div style={{ minHeight: 400, marginBottom: 20 }}>
        {messages.map((msg, i) => (
          <div key={i} style={{
            padding: '8px 12px',
            margin: '8px 0',
            borderRadius: 8,
            background: msg.role === 'user' ? '#e3f2fd' : '#f5f5f5',
            textAlign: msg.role === 'user' ? 'right' : 'left'
          }}>
            {msg.content}
          </div>
        ))}
        {currentContent && (
          <div style={{ padding: '8px 12px', background: '#f5f5f5', borderRadius: 8 }}>
            {currentContent}
            <span style={{ animation: 'blink 1s infinite' }}>▊</span>
          </div>
        )}
      </div>

      <form onSubmit={handleSubmit} style={{ display: 'flex', gap: 8 }}>
        <input
          value={input}
          onChange={(e) => setInput(e.target.value)}
          placeholder="输入消息..."
          disabled={isStreaming}
          style={{ flex: 1, padding: 8 }}
        />
        {isStreaming ? (
          <button type="button" onClick={stopStreaming}>停止</button>
        ) : (
          <button type="submit">发送</button>
        )}
      </form>
    </div>
  );
}

生产环境踩坑实录

坑1：Nginx 缓冲导致流式失效

# ❌ 默认Nginx配置会缓冲响应，导致内容攒一大堆才一起发
# 用户看到的是"等了5秒然后刷出一大段"，而不是逐字出现

# ✅ 必须禁用缓冲
location /api/chat/stream {
    proxy_pass http://localhost:3000;

    # 关键配置 —— 缺一不可
    proxy_buffering off;          # 禁用代理缓冲
    proxy_cache off;              # 禁用缓存
    proxy_http_version 1.1;       # 使用HTTP/1.1
    proxy_set_header Connection '';
    chunked_transfer_encoding off; # 禁用分块传输编码

    # 超时设置（AI响应可能很长）
    proxy_read_timeout 300s;
    proxy_send_timeout 300s;
}

本地开发一切正常，部署到服务器后流式效果消失。最后发现是 Nginx 的 proxy_buffering 默认是开的。

坑2：前端渲染性能

// ❌ 每收到一个字符就 setState —— 大量重渲染，页面卡顿
setContent(prev => prev + chunk);  // 每秒可能触发 20-50 次渲染

// ✅ 节流渲染 —— 累积一批再更新
const bufferRef = useRef('');
const rafRef = useRef(null);

function appendContent(chunk) {
  bufferRef.current += chunk;

  // 使用 requestAnimationFrame 合并更新
  if (!rafRef.current) {
    rafRef.current = requestAnimationFrame(() => {
      setContent(bufferRef.current);
      rafRef.current = null;
    });
  }
}

小结

SSE 的未来：会被 Streamable HTTP 取代吗？

Streamable HTTP：它是一种”按需升级”的 HTTP 通信方式——简单请求用普通 HTTP 响应，复杂请求自动升级为 SSE 流式响应，全部走一个端点

2025年3月26日，MCP 协议将传输层从 HTTP+SSE 升级为 Streamable HTTP。这是不是意味着 SSE 过时了？

我的理解：不是。Streamable HTTP 本质上是 SSE 的升级版——它保留了 SSE 的流式能力，但解决了 SSE 的几个痛点：

  1. SSE 要求长连接 → Streamable HTTP 支持无状态模式
  2. SSE 需要专用端点 → Streamable HTTP 统一端点
  3. SSE 重连不够优雅 → Streamable HTTP 内置会话管理

但对于 AI 聊天应用来说，SSE 仍然是最佳方案。Streamable HTTP 更多是为 MCP 这类”工具协议”场景设计的。

SSE 核心要点：

  1. AI 应用必须上流式输出——用户体验差异是量级级的
  2. SSE 是最佳选择——单向推送、基于HTTP、自动重连
  3. 生产环境注意代理缓冲——Nginx 的缓冲是最大的坑
  4. 前端注意渲染性能——不要每个字符都触发一次渲染
