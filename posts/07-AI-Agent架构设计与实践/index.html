什么是 AI Agent

从 LLM 到 Agent 的本质跃迁

LLM：被动响应
  用户提问 → LLM生成回答 → 结束
  特点：一轮对话，无规划能力

Agent：主动执行
  用户给出目标 → Agent理解意图 → 制定计划 → 执行动作 →
  观察结果 → 调整策略 → 继续执行 → 达成目标
  特点：多轮循环，自主决策

Agent 的本质是给 LLM 加上了一个**”感知-决策-执行-反馈”的循环**。LLM 提供”思考”能力，工具提供”执行”能力，循环提供”持续”能力。

Agent 四大核心组件

一个完整的 Agent 由四大核心组件构成：

                        ┌─────────────────────────┐
                        │      🧠 LLM（大脑）       │
                        │   理解、推理、生成、决策    │
                        └────────────┬────────────┘
                                     │
          ┌──────────────────────────┼──────────────────────────┐
          │                          │                          │
  ┌───────┴───────┐         ┌───────┴───────┐         ┌───────┴───────┐
  │  📋 规划系统   │         │  🧲 记忆系统   │         │  🔧 工具系统   │
  │  Planning     │         │  Memory       │         │  Tools        │
  │               │         │               │         │               │
  │ · 任务分解    │         │ · 短期记忆     │         │ · 工具注册    │
  │ · 思维链推理  │         │  (对话上下文)  │         │ · 动态发现    │
  │ · 自我反思    │         │ · 长期记忆     │         │ · 能力扩展    │
  │ · 策略调整    │         │  (向量数据库)  │         │ · 权限控制    │
  └───────┬───────┘         └───────┬───────┘         └───────┬───────┘
          │                          │                          │
          └──────────────────────────┼──────────────────────────┘
                                     │
                            ┌────────┴────────┐
                            │  ⚡ 行动系统     │
                            │  Action         │
                            │                 │
                            │ · 执行工具调用   │
                            │ · 处理返回结果   │
                            │ · 错误恢复重试   │
                            │ · 超限保护熔断   │
                            └─────────────────┘

这四个组件就像人类完成任务的过程——先规划怎么做，凭借记忆中的知识和经验，使用合适的工具，然后行动执行。缺少任何一个，Agent 都不完整。

1. 规划系统（Planning）—— Agent 的”项目经理”

规划是 Agent 区别于普通 LLM 最关键的能力。没有规划，Agent 只是一个会调工具的聊天机器人。让 Agent 在行动前”想清楚”，而不是冲动行事。

2. 记忆系统（Memory）—— Agent 的”大脑存储”

记忆决定了 Agent 能否从经验中学习、避免重复犯错。

记忆系统架构：

┌─────────────────────────────────────────────┐
│                  记忆系统                     │
├──────────────────┬──────────────────────────┤
│    短期记忆       │       长期记忆            │
│  Short-Term      │     Long-Term            │
│                  │                          │
│  · 当前对话历史   │  · 向量数据库存储          │
│  · 执行步骤记录   │  · 语义检索历史经验        │
│  · 工具返回结果   │  · 跨会话知识积累          │
│                  │                          │
│  实现：消息数组   │  实现：Embedding + 向量DB  │
│  生命周期：单次任务│  生命周期：持久化           │
└──────────────────┴──────────────────────────┘

为什么记忆系统如此重要？ 没有短期记忆，Agent 不知道上一步做了什么；没有长期记忆，Agent 永远无法从经验中成长。

3. 工具系统（Tools）—— Agent 的”双手”

LLM 只能”想”，工具让 Agent 能”做”。工具系统决定了 Agent 能力的边界。

// tools.mjs —— 工具注册与管理系统
class ToolRegistry {
  constructor() {
    this.tools = new Map();
  }

  // 注册工具（附带元数据，供 LLM 理解工具能力）
  register(name, config) {
    this.tools.set(name, {
      name,
      description: config.description,
      parameters: config.parameters,
      handler: config.handler,
      // 元数据：帮助 Agent 更好地选择工具
      meta: {
        category: config.category || 'general',    // 工具分类
        riskLevel: config.riskLevel || 'low',       // 风险等级：low/medium/high
        rateLimitPerMin: config.rateLimit || 60,    // 调用频率限制
        requiresApproval: config.requiresApproval || false  // 是否需要人工确认
      }
    });
  }

  // 获取所有工具定义（转为 OpenAI Function Calling 格式）
  getToolDefinitions() {
    return Array.from(this.tools.values()).map(tool => ({
      type: 'function',
      function: {
        name: tool.name,
        description: tool.description,
        parameters: tool.parameters
      }
    }));
  }

  // 按分类过滤工具（缩小 Agent 的行动空间，提升准确率）
  getToolsByCategory(category) {
    return Array.from(this.tools.values())
      .filter(t => t.meta.category === category);
  }

  // 执行工具（带权限检查）
  async execute(name, args) {
    const tool = this.tools.get(name);
    if (!tool) throw new Error(`未知工具: ${name}`);

    // 高风险工具需要人工确认
    if (tool.meta.requiresApproval) {
      console.log(`⚠️ 高风险操作 [${name}]，需要人工确认`);
      console.log(`   参数: ${JSON.stringify(args)}`);
      // 实际项目中这里会暂停等待用户确认
    }

    return await tool.handler(args);
  }
}

const registry = new ToolRegistry();

  description: '搜索互联网获取信息',

  riskLevel: 'low',
  parameters: {
    type: 'object',
    properties: {
      query: { type: 'string', description: '搜索关键词' }
    },
    required: ['query']
  },
  handler: async ({ query }) => {
    // 调用搜索API
    return `搜索"${query}"的结果: ...`;
  }
});

// 执行类工具（中风险）
registry.register('run_code', {
  description: '在沙箱中执行代码',
  category: 'execution',
  riskLevel: 'medium',
  parameters: {
    type: 'object',
    properties: {
      language: { type: 'string', enum: ['javascript', 'python'] },
      code: { type: 'string', description: '要执行的代码' }
    },
    required: ['language', 'code']
  },
  handler: async ({ language, code }) => {
    // 在沙箱中执行
    return `执行结果: ...`;
  }
});

// 操作类工具（高风险，需要人工确认）
registry.register('send_email', {
  description: '发送邮件',
  category: 'action',
  riskLevel: 'high',
  requiresApproval: true,    // 发邮件必须人工确认！
  parameters: {
    type: 'object',
    properties: {
      to: { type: 'string', description: '收件人' },
      subject: { type: 'string', description: '主题' },
      body: { type: 'string', description: '正文' }
    },
    required: ['to', 'subject', 'body']
  },
  handler: async ({ to, subject, body }) => {
    return `邮件已发送至 ${to}`;
  }
});

工具设计的关键原则：

  • 少即是多：每个场景 3-5 个工具足够，工具太多 LLM 反而选不准
  • 描述决定一切：LLM 是通过 description 来理解工具用途的，描述不准，调用就不准
  • 分级控制：信息查询类自动执行，数据修改类需要确认，不可逆操作必须人工审批

4. 行动系统（Action）—— Agent 的”执行引擎”

行动系统负责将规划转化为实际操作，并处理执行过程中的各种异常：

// action.mjs —— 行动执行引擎
class ActionEngine {
  constructor(toolRegistry) {
    this.toolRegistry = toolRegistry;
    this.guard = new ExecutionGuard({
      maxSteps: 20,
      maxTokens: 50000,
      maxTime: 120000   // 2分钟超时
    });
  }

  // 执行单个工具调用（带重试和错误恢复）
  async executeToolCall(toolCall, maxRetries = 2) {
    const funcName = toolCall.function.name;
    const args = JSON.parse(toolCall.function.arguments);

    for (let attempt = 0; attempt <= maxRetries; attempt++) {
      try {
        console.log(`🔧 执行: ${funcName}(${JSON.stringify(args)})`);
        const result = await this.toolRegistry.execute(funcName, args);
        console.log(`👁️ 结果: ${result}`);

        return { success: true, result };

      } catch (error) {
        console.warn(`⚠️ 第${attempt + 1}次失败: ${error.message}`);

        if (attempt === maxRetries) {
          return {
            success: false,
            error: `工具执行失败(已重试${maxRetries}次): ${error.message}`
          };
        }

        // 指数退避重试
        await new Promise(r => setTimeout(r, 1000 * Math.pow(2, attempt)));
      }
    }
  }

  // 批量执行（支持并行，提升效率）
  async executeBatch(toolCalls) {
    // 检查工具间是否有依赖关系
    const independent = this.findIndependentCalls(toolCalls);
    const dependent = toolCalls.filter(tc => !independent.includes(tc));

    // 无依赖的工具并行执行
    const parallelResults = await Promise.all(
      independent.map(tc => this.executeToolCall(tc))
    );

    // 有依赖的工具串行执行
    const serialResults = [];
    for (const tc of dependent) {
      serialResults.push(await this.executeToolCall(tc));
    }

    return [...parallelResults, ...serialResults];
  }

  findIndependentCalls(toolCalls) {
    // 简化逻辑：不同名工具视为独立
    const seen = new Set();
    return toolCalls.filter(tc => {
      const name = tc.function.name;
      if (seen.has(name)) return false;
      seen.add(name);
      return true;
    });
  }
}

// 执行超限保护，避免无限循环
class ExecutionGuard {
  constructor(config) {
    this.maxSteps = config.maxSteps || 20;
    this.maxTokens = config.maxTokens || 50000;
    this.maxTime = config.maxTime || 120000;
    this.startTime = Date.now();
    this.totalTokens = 0;
  }

  check(step, tokensUsed) {
    this.totalTokens += tokensUsed;

    if (step >= this.maxSteps) return { stop: true, reason: '步数超限' };
    if (this.totalTokens >= this.maxTokens) return { stop: true, reason: 'Token预算耗尽' };
    if (Date.now() - this.startTime >= this.maxTime) return { stop: true, reason: '执行超时' };

    return { stop: false };
  }
}

规划系统决定调用哪个工具，行动系统负责可靠地执行它——包括重试、超时、并行优化等工程问题。

四大组件协作流程

把四大组件串起来，一个完整的 Agent 执行流程如下：

用户输入: "帮我查一下今年Q1的销售数据，生成趋势图表，然后发邮件给老板"

📋 规划系统：
  ├── Step 1: 查询数据库获取Q1销售数据          [tool: query_database]
  ├── Step 2: 对数据进行统计分析                [tool: run_code]
  ├── Step 3: 生成趋势图表                     [tool: generate_chart]
  └── Step 4: 发送邮件（需人工确认）             [tool: send_email]

🧲 记忆系统：
  ├── 短期：记录每步的执行结果，供后续步骤使用
  └── 长期：检索到"上次老板要求图表用蓝色主题"

🔧 工具系统：
  ├── query_database → 返回Q1数据
  ├── run_code → 执行Python统计脚本
  ├── generate_chart → 生成蓝色主题图表（基于长期记忆）
  └── send_email → 等待人工确认后发送

⚡ 行动系统：
  ├── Step 1-3：自动执行（低风险）
  ├── Step 4：暂停，等待人工确认（高风险）
  ├── 如果 Step 1 失败：重试3次，仍失败则触发重规划
  └── 如果总耗时 > 2分钟：熔断，返回已完成的部分结果

ReAct 模式：Agent 的经典架构

ReAct（Reasoning + Acting）是最经典、最实用的 Agent 架构：

Think（思考）→ Act（行动）→ Observe（观察）→ Think... 循环

可运行的 ReAct Agent 实现

// react-agent.mjs —— 一个可运行的 ReAct Agent
import OpenAI from 'openai';

const openai = new OpenAI({ apiKey: process.env.OPENAI_API_KEY });

// ========== 工具定义 ==========
const toolImplementations = {
  calculate: ({ expression }) => {
    try {
      // 安全的数学表达式求值
      const result = Function(`"use strict"; return (${expression})`)();
      return `计算结果: ${expression} = ${result}`;
    } catch (e) {
      return `计算错误: ${e.message}`;
    }
  },

  search_web: async ({ query }) => {
    // 模拟搜索结果（实际项目中调用搜索API）
    const mockResults = {
      'JavaScript创建时间': 'JavaScript由Brendan Eich在1995年创建',
      'Node.js发布时间': 'Node.js由Ryan Dahl在2009年首次发布',
    };
    for (const [key, value] of Object.entries(mockResults)) {
      if (query.includes(key) || key.includes(query)) return value;
    }
    return `搜索"${query}"的结果：暂无找到明确信息`;
  },

  get_current_time: () => {
    return `当前时间: ${new Date().toLocaleString('zh-CN')}`;
  }
};

const tools = [
  {
    type: 'function',
    function: {
      name: 'calculate',
      description: '执行数学计算。当需要做算术运算时使用。',
      parameters: {
        type: 'object',
        properties: {
          expression: { type: 'string', description: '数学表达式，如 "2+3*4"' }
        },
        required: ['expression']
      }
    }
  },
  {
    type: 'function',
    function: {
      name: 'search_web',
      description: '搜索互联网获取信息。当需要查找事实性知识时使用。',
      parameters: {
        type: 'object',
        properties: {
          query: { type: 'string', description: '搜索关键词' }
        },
        required: ['query']
      }
    }
  },
  {
    type: 'function',
    function: {
      name: 'get_current_time',
      description: '获取当前时间。',
      parameters: { type: 'object', properties: {} }
    }
  }
];

// ========== Agent 核心循环 ==========
async function runAgent(goal, maxSteps = 10) {
  console.log(`🎯 目标: ${goal}\n${'='.repeat(50)}`);

  const messages = [
    {
      role: 'system',
      content: `你是一个智能助手，通过"思考-行动-观察"循环来完成任务。

每一步你需要：
1. 思考：分析当前状态，决定下一步做什么
2. 行动：选择合适的工具执行
3. 观察：根据工具返回结果，决定是否需要继续

当你认为任务已完成，直接给出最终答案，不再调用工具。
保持步骤精简，避免不必要的工具调用。`
    },
    { role: 'user', content: goal }
  ];

  for (let step = 1; step <= maxSteps; step++) {
    console.log(`\n--- 步骤 ${step} ---`);

    const response = await openai.chat.completions.create({
      model: 'gpt-4',
      messages,
      tools,
      tool_choice: 'auto'
    });

    const message = response.choices[0].message;
    messages.push(message);

    // 如果没有工具调用，说明Agent认为任务完成了
    if (!message.tool_calls) {
      console.log(`\n💡 思考: ${message.content?.substring(0, 100)}`);
      console.log(`\n✅ 最终答案:\n${message.content}`);
      return message.content;
    }

    // 执行工具调用
    for (const toolCall of message.tool_calls) {
      const funcName = toolCall.function.name;
      const args = JSON.parse(toolCall.function.arguments);

      console.log(`🔧 行动: ${funcName}(${JSON.stringify(args)})`);

      const impl = toolImplementations[funcName];
      let result;
      try {
        result = typeof impl === 'function'
          ? await impl(args)
          : `未知工具: ${funcName}`;
      } catch (e) {
        result = `执行错误: ${e.message}`;
      }

      console.log(`👁️ 观察: ${result}`);

      messages.push({
        role: 'tool',
        tool_call_id: toolCall.id,
        content: String(result)
      });
    }
  }

  return '达到最大步数限制，任务未完成';
}

// ========== 测试 ==========
// 简单任务：一步完成
await runAgent('现在几点了？');

// 复杂任务：需要多步推理
await runAgent('JavaScript是哪年创建的？距今多少年了？请计算。');

// 输出示例：
// 🎯 目标: JavaScript是哪年创建的？距今多少年了？请计算。
// ==================================================
//
// --- 步骤 1 ---
// 🔧 行动: search_web({"query":"JavaScript创建时间"})
// 👁️ 观察: JavaScript由Brendan Eich在1995年创建
//
// --- 步骤 2 ---
// 🔧 行动: get_current_time({})
// 👁️ 观察: 当前时间: 2025/3/25 10:30:00
//
// --- 步骤 3 ---
// 🔧 行动: calculate({"expression":"2025 - 1995"})
// 👁️ 观察: 计算结果: 2025 - 1995 = 30
//
// --- 步骤 4 ---
// ✅ 最终答案:
// JavaScript由Brendan Eich在1995年创建，距今已有30年了。

Multi-Agent：多智能体协作

单个 Agent 能力有限，复杂任务需要多个专业 Agent 协作：

// multi-agent.mjs —— 多Agent协作示例
class MultiAgentTeam {
  constructor() {
    this.agents = new Map();
  }

  // 注册Agent
  register(name, config) {
    this.agents.set(name, {
      name,
      role: config.role,
      systemPrompt: config.systemPrompt,
      tools: config.tools || []
    });
  }

  // 协调执行
  async execute(task) {
    console.log(`📋 团队任务: ${task}\n`);

    // 1. Planner 分解任务
    const plan = await this.callAgent('planner', `请分解以下任务为具体步骤:${task}`);
    console.log(`📝 执行计划:${plan}`);

    // 2. 按计划分配给专业Agent
    const results = {};
    results.analysis = await this.callAgent('analyst', `分析以下需求:${task}`);
    results.implementation = await this.callAgent('developer', `基于分析结果实现:${results.analysis}`);
    results.review = await this.callAgent('reviewer', `审查以下实现:${results.implementation}`);

    // 3. 汇总结果
    return results;
  }

  async callAgent(name, prompt) {
    const agent = this.agents.get(name);
    if (!agent) throw new Error(`Agent "${name}" not found`);

    console.log(`🤖 [${agent.role}] 执行中...`);

    const response = await openai.chat.completions.create({
      model: 'gpt-4',
      messages: [
        { role: 'system', content: agent.systemPrompt },
        { role: 'user', content: prompt }
      ]
    });

    const result = response.choices[0].message.content;
    console.log(`   ✅ 完成`);
    return result;
  }
}

// 注册团队成员
const team = new MultiAgentTeam();

team.register('planner', {
  role: '项目经理',
  systemPrompt: '你是项目经理，负责将复杂任务分解为清晰的执行步骤。每个步骤要有明确的目标和负责人。'
});

team.register('analyst', {
  role: '需求分析师',
  systemPrompt: '你是需求分析师，擅长分析需求、识别关键问题、提出技术方案。'
});

team.register('developer', {
  role: '开发工程师',
  systemPrompt: '你是高级开发工程师，根据需求分析编写高质量的代码实现。'
});

team.register('reviewer', {
  role: '代码审查员',
  systemPrompt: '你是代码审查专家，从代码质量、安全性、性能角度审查代码，给出具体的改进建议。'
});

// 执行
await team.execute('开发一个用户登录功能，支持邮箱和手机号登录');

Agent 设计原则

原则1：越简单越好

// ❌ 过度设计：复杂的10层Agent架构
// 大部分时候，一个简单的 ReAct 循环就够了

// ✅ 简单有效：先用最简单的架构，遇到瓶颈再升级

async function executeWithApproval(plan) {
  console.log('📋 执行计划:', plan);
  const approved = await askUser('是否执行此计划？(y/n)');

  if (approved) {
    return await agent.execute(plan);
  } else {
    return await agent.revise(plan, '用户拒绝，请修改计划');
  }
}

小结

我在实际项目中发现，Agent 最大的问题不是”能不能做”，而是”做得稳不稳”。LLM 的非确定性本质（同一个输入可能产生不同输出）让 Agent 在复杂场景下表现极不稳定。

我的应对策略：

  1. 缩小行动空间：不给 Agent 太多工具选择，每个场景 3-5 个足够
  2. 结构化输出：用 JSON Schema 约束 LLM 输出格式
  3. 分级执行：简单操作自动执行，高风险操作人工确认
  4. 兜底机制：Agent 失败后，优雅回退到传统流程

Agent 核心要点：

  1. 四大核心缺一不可——规划（Planning）、记忆（Memory）、工具（Tools）、行动（Action）构成完整 Agent
  2. 规划是灵魂——没有规划能力的 Agent 只是一个会调工具的聊天机器人
  3. 记忆分两层——短期记忆（对话上下文）保证连贯性，长期记忆（向量数据库）实现经验积累
  4. 工具要分级——信息查询自动执行，数据修改需确认，不可逆操作必须人工审批
  5. 行动要可靠——重试、超时、熔断是生产环境的必需品
  6. ReAct 是最实用的架构——思考、行动、观察循环，覆盖 80% 的 Agent 场景
  7. 简单优于复杂——先用最简单的方案，按需升级
