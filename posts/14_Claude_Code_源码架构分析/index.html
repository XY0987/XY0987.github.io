📦 Claude Code 源码全景分析

项目概述

这是 Claude Code —— Anthropic 的 AI 编程助手 CLI 工具的完整源码。它是一个基于终端的交互式 AI 编程工具，用户在终端中与 Claude 模型对话，Claude 可以读写文件、执行命令、搜索代码库，完成各种软件开发任务。

技术栈

  • 语言: TypeScript (~1900 文件)
  • 运行时: Bun (带 bun:bundle 特性门控和死代码消除)
  • UI 框架: React + Ink (深度定制的终端 UI 框架)
  • AI 模型: Anthropic Claude API (流式调用 + 工具使用)
  • 构建: Bun bundler (feature flag 条件编译)
  • 包管理: npm/pnpm

12. 循环决策与恢复 query.ts 循环尾部 根据 API 返回的 stop_reason 和当前状态决定：① tool_use → 收集所有工具结果 → 注入附件（内存预取 pendingMemoryPrefetch、技能发现 pendingSkillPrefetch、队列命令 getCommandsByMaxPriority）→ 刷新 MCP tools → continue 回到循环顶部 → ② end_turn → handleStopHooks() 检查停止钩子（自定义 hooks 可 block 并 retry）→ 返回 Terminal → ③ max_output_tokens → escalate 到 64k → 最多 3 次多轮恢复（MAX_OUTPUT_TOKENS_RECOVERY_LIMIT）→ ④ prompt_too_long (413) → context collapse drain → reactive compact → 错误上报 → ⑤ token budget → checkTokenBudget() 检查 +500k 自动续接 → ⑥ max_turns → 达到上限退出 Terminal（结束）或 Continue（继续循环，附带原因）

核心引擎详解

src/query.ts (1730行) — 核心查询循环

这是 Claude Code 最复杂的文件之一，实现了 模型调用 → 工具执行 → 继续循环 的核心机制：

  • query(): 异步生成器，对外主 API
  • queryLoop(): 内部 while(true) 循环，每次迭代包含：
    1. Snip compaction: 裁剪历史消息
    2. Microcompact: 微压缩（缓存编辑）
    3. Context collapse: 上下文折叠
    4. Autocompact: 自动压缩（token 接近上限时）
    5. API 调用: deps.callModel() 流式调用模型
    6. Streaming tool execution: 流式工具执行（边接收边执行）
    7. Fallback model: 模型回退机制
    8. Stop hooks: 停止钩子检查
    9. Max output tokens recovery: 输出 token 限制恢复（最多 3 次）
    10. Reactive compact: 响应式压缩（413 错误恢复）

核心设计理念

1. 极致的启动性能优化

  • 副作用严格排序 (profileCheckpoint → startMdmRawRead → startKeychainPrefetch)
  • I/O 并行化（MDM 子进程、Keychain 读取并行）
  • 延迟导入（feature() + 动态 import() + require()）
  • bun:bundle 的 feature() 实现死代码消除

2. Fail-closed 安全模型

  • 多层权限检查：deny rules → tool permissions → hooks → 用户确认
  • buildTool() 默认值：isConcurrencySafe=false、isReadOnly=false
  • 工作区信任对话 (TrustDialog)
  • CLAUDE.md 外部包含审批
  • --dangerously-skip-permissions 的容器/网络安全检查

3. 弹性恢复的查询循环

  • prompt-too-long → context collapse drain → reactive compact → 错误上报
  • max_output_tokens → escalate to 64k → multi-turn recovery (最多 3 次)
  • 模型回退 (fallback model) 机制
  • streaming abort 处理
  • stop hook blocking → retry

4. 双模式运行

  • 交互式 REPL: React/Ink TUI，完整终端交互
  • Headless SDK: QueryEngine，适用于 CI/CD、cowork、desktop 集成

5. 四层扩展机制

  1. Tool — 模型可调用的原子操作 (Bash, Read, Edit…)
  2. Command — 用户斜杠命令 (/compact, /resume, /config…)
  3. Skill — Prompt 模板命令 (bundled + 用户自定义)
  4. Plugin — 外部插件 (MCP servers, hooks, agents, commands)

6. Feature Flag 条件编译

  • feature('XXX') + bun:bundle 实现按环境剪裁代码
  • 内部版本 (USER_TYPE=ant) 和外部版本共享同一代码库
  • INTERNAL_ONLY_COMMANDS 在外部构建中被消除
  • 条件 require() 确保未使用的模块不被加载

7. 状态管理双层架构

  • bootstrap/state.ts — 全局 singleton，进程级状态 (session ID, CWD, 遥测计数器, OTel meter)
  • AppState (React Context) — 组件级状态 (权限、MCP 客户端、工具、消息)
