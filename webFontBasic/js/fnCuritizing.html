函数柯里化，实现add(1)(2)...累加函数

什么是柯里化

函数柯里化是一种将接受多个参数的函数转换为一系列接受单个参数的函数的过程。这种转换过程使得函数更加灵活，可以更方便地进行部分应用和复用。

简单来说就是将多个参数分开传递，最终返回结果

实现add(1)(2)(3)

普通函数

cosnt add=(a,b,c)=>{
  return a+b+c
}

柯里化函数

const add=(a)=>{
  return (b) => {
    return (c) => {
      return a + b + c
    }
  }
}

递归实现

function curitizing(fn, ...args) {
  // 获取fn函数的接受参数个数
  const len = fn.length;
  return function () {
    // 箭头函数没有arguments
    const newArgs = args.concat(Array.prototype.slice.call(arguments));
    // 小于函数的要求的传参个数
    if (newArgs.length < len) {
      return curitizing.call(this, fn, ...newArgs);
    } else {
      return fn.call(this, ...newArgs);
    }
  };
}
const fn = (a, b, c) => {
  return a * b * c;
};
const foo = curitizing(fn);
console.log(foo(1, 2, 3));
console.log(foo(1, 2)(3));
console.log(foo(1)(2)(3));

上述方法有个缺点就是，他只能接受fn函数要求的参数个数，也只处理了参数少于函数要求参数的情况

改进版

function add(...args) {
  return args.reduce((prev, next) => prev + next);
}
function currying(fn) {
  let argArr = [];
  return function temp(...newArgs) {
    // 知道不出入值才会返回新值
    if (newArgs.length) {
      argArr = [...argArr, ...newArgs];
      return temp;
    } else {
      let val = fn.apply(this, argArr);
      args = [];
      return val;
    }
  };
}

const addCurry = currying(add);
console.log(addCurry(1)(2)());

柯里化的优缺点

优点

  • 参数复用，每一步只处理一个参数，更加清晰
  • 函数更加灵活
  • 拓展性高，拆解成各个函数，根据参数不同，可以组合的可能性就越多

缺点

  • 如果参数过多，看起来比较绕
  • 应用了闭包，可能会导致内存泄漏

柯里化的简单应用

我们在访问接口的时候经常会有传递多个参数的，也有很多情况下，传递的参数都是相对固定的（比如说header里边传的参数）我们就可以使用函数柯里化，把固定的参数给单独拆分出来，去传递不固定的参数

const defaultPrefix = 'XXXX'
const curry = (fn)=>{
    //传入的fn的参数个数
    const fnLen = fn.length
    const innerFun = (...params)=>{
    //如果传入的参数个数少于原函数的参数个数
    //则返回当前函数，并将下一个函数的参数与当前函数的参数合并作为返回函数的参数，继续判断
        if(params.length <fnLen){
            return (...selfParam)=>innerFun(...params,...selfParam)
        }else{
    //如果传入的参数个数等于原函数的参数个数，执行原函数
            return fn(...params)
        }
    }
    return innerFun
}

const coreRequest = curry((method,options,prefix,url,data)=>{
    const dataSet = {}
    method.toLowerCase() == 'get' ? dataSet.params = data : dataSet.data = data
    return request({
        url:prefix+url,
        method,
        ...dataSet,
        ...options
    })
})
//常规json格式请求
const jsonContentType = {
    headers:{
        'Content-Type':'application/json;charset=utf-8'
    }
}
//返回带第一个method参数的函数
const post = coreRequest('post')
const get = coreRequest('get')
//返回请求头部是application/json的post请求函数
const postJson = post(jsonContentType)
//返回请求头部是application/json的get请求函数
const getJson = get(jsonContentType)
//返回带默认服务前缀的post请求
const postApi = postJson(defaultPrefix)
